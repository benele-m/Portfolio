Real problems, real fixes

Troubleshooting Experience

A few real situations I've worked through — each broken down by problem, cause, solution, and outcome.

Hardware · Windows

Blue Screen of Death (BSOD)

  • Problem

    Windows desktop crashed with a BSOD multiple times a day, often during heavier workloads like compiling or gaming.

  • Cause

    Memory diagnostics revealed a faulty RAM stick, made worse by an outdated graphics driver causing additional kernel faults under load.

  • Solution

    Ran MemTest86 to confirm the failing module, removed it, reseated the remaining stick, then performed a clean graphics driver reinstall.

  • Outcome

    System has been crash-free for months. Learned how to read stop codes, use Event Viewer, and diagnose hardware vs. software faults systematically.

Backend · Database

Database Connection Timeouts

  • Problem

    A small web app started returning 500 errors under modest load. API requests hung for several seconds before timing out.

  • Cause

    The PostgreSQL connection pool was exhausted — long-running queries were holding connections, and new requests had nowhere to go.

  • Solution

    Tuned pool size and idle timeouts, added query timeouts, created a missing index on the hot table, and ensured connections were released in finally blocks.

  • Outcome

    Average response time dropped sharply and timeouts disappeared. Got a much deeper feel for connection pooling and how query plans interact with load.

Networking · LAN

Random LAN Disconnects

  • Problem

    Multiple devices on a home network kept losing internet at seemingly random times, especially when new devices joined.

  • Cause

    A misconfigured DHCP scope on the router was handing out conflicting IPs, and the lease time was too short for the number of devices.

  • Solution

    Reset the DHCP scope to a clean range, extended lease times, and assigned static reservations to the printer, NAS, and main workstation.

  • Outcome

    Network has been stable since. Walked away with a much clearer mental model of DHCP, ARP, and how home routers actually work under the hood.

Cybersecurity · Server

SSH Brute-Force on a Personal Server

  • Problem

    Logs on a personal Linux server were filling up with thousands of failed SSH login attempts from random IPs each day.

  • Cause

    SSH was open on the default port, with password authentication enabled and no rate limiting in place.

  • Solution

    Switched to key-based authentication, disabled root login and password auth in sshd_config, installed fail2ban, and locked things down at the firewall level.

  • Outcome

    Brute-force noise disappeared from the logs. Got real, hands-on understanding of basic server hardening instead of only reading about it in class.

Web Development

CORS Error on COntact form (Cross-Origin Request Blocked)

  • Problem

    Cross-Origin Request Blocked: The Same Origin Policy disallows reading the remote resource at https://cdnjs.cloudflare.com/ajax/libs/font-awesome/6.5.1/css/all.min.css. (Reason: CORS request did not succeed). Status code: (null).

  • Cause

    I was running the project locally using file://path instead of a proper local server. This caused the browser to treat the request as a cross-origin, which blocked the form submission for secutity reasons.

  • Solution

    Used live server to run project.

  • Outcome
